# 📦 GitHub 部署手册 · AI 产业新闻每日自动更新

> 目标：把持仓看板部署到 GitHub Pages，让「AI 产业带重点新闻」每天自动更新，
> 同时**你的交易/保险明细绝不公开**。

---

## ✅ 隐私已处理（重要）

本项目已做隐私隔离，你可以放心推公开仓库：

| 文件 | 是否上传 GitHub | 内容 |
|------|----------------|------|
| `index.html` | ✅ 上传 | 页面框架（**已清空交易/保险明细**） |
| `news.json` | ✅ 上传 | AI 产业新闻（公开信息） |
| `scripts/fetch-news.js` | ✅ 上传 | 抓新闻脚本 |
| `.github/workflows/update-news.yml` | ✅ 上传 | 每日定时任务 |
| **`private-data.js`** | 🔒 **不上传** | **你的真实交易 + 保险明细** |
| `.gitignore` | ✅ 上传 | 已配置忽略 private-data.js |

> 📌 `private-data.js` 已写入 `.gitignore`，`git` 会自动跳过它。
> 部署后别人访问网址：**只看到新闻，看不到你任何交易**。
> 你本地打开：`private-data.js` 存在 → 完整数据照常显示。

## 🚀 部署步骤（一次性，约 20 分钟）

### 第 1 步：安装 Git（若已装跳过）
Mac 打开「终端」，输入 `git --version`，没报错就说明已装。

### 第 2 步：注册 GitHub
访问 https://github.com → Sign up，注册一个账号。

### 第 3 步：新建仓库
1. 登录后点右上角 `+` → `New repository`
2. Repository name：填 `stella-portfolio`
3. 选 **Public**（公开仓库，Actions + Pages 才免费）
4. 点 `Create repository`

### 第 4 步：上传代码
在终端依次执行（把 `你的用户名` 换成实际 GitHub 用户名）：

```bash
cd /Users/sj/WorkBuddy/20260505115358/stella-portfolio
git init
git add .
git commit -m "init: 持仓看板 + AI 新闻自动更新"
git branch -M main
git remote add origin https://github.com/你的用户名/stella-portfolio.git
git push -u origin main
```

> ⚠️ 推送前确认：`git status` 里**不应出现 private-data.js**（被 .gitignore 挡住了才对）。
> 可执行 `git check-ignore private-data.js` 验证，输出文件名即表示已忽略。

### 第 5 步：开启 GitHub Pages
1. 仓库页面 → `Settings` → 左侧 `Pages`
2. Source 选 `Deploy from a branch`
3. Branch 选 `main` + `/ (root)` → `Save`
4. 等 1-2 分钟，页面顶部出现网址：
   `https://你的用户名.github.io/stella-portfolio/`

### 第 6 步：开启每日自动更新
1. 仓库页面 → `Actions` 标签
2. 若提示是否启用 workflow → 点 `I understand my workflows, enable them`
3. 找到「每日更新 AI 产业新闻」→ 点 `Run workflow` 手动测一次
4. 之后每天北京时间 **08:00** 自动跑，抓新闻 → 更新 news.json → 网页自动显示

## 🔄 日常使用

| 场景 | 操作 |
|------|------|
| 看新闻 | 直接访问 Pages 网址，每天自动更新 |
| 记自己的交易 | **在本地** `file://` 打开 index.html，数据存本地浏览器 |
| 换设备看持仓 | 用「关联备份 / 从备份文件恢复」搬 private-data 或备份 json |
| 手动更新新闻 | 直接编辑 news.json 后 `git push`，或去 Actions 手动 Run |
| 调整抓取源/关键词 | 改 `scripts/fetch-news.js` 里的 FEEDS / LAYER_KEYWORDS |

## ⚠️ 注意事项

1. **RSS 源会变动**：脚本用了 rsshub / 36氪等公开源，若某天抓不到，脚本会**保留原新闻不清空**，页面不会变空。可随时在 fetch-news.js 里换更稳定的源。
2. **Pages 是公开的**：任何人拿到网址都能访问（但只有新闻，无隐私）。若想加密码，需换付费方案。
3. **本地记账 vs 线上看新闻**：这是刻意的分离设计——隐私数据只在你本地，线上只放公开的新闻框架。
4. **首次 Actions 可能失败**：若 rsshub 公共实例限流，多试几次或换源即可。
